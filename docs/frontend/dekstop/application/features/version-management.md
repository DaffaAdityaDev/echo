## Deskripsi

The desktop application uses `electron-updater` with a generic provider to enable automatic updates. Update configuration is split between `electron-builder.yml` (packaging-time) and `dev-app-update.yml` (development-time) to control where the app checks for new versions.

## File Structure

```
frontend/dekstop/
├── electron-builder.yml        # Build config with publish block
├── dev-app-update.yml          # Dev-mode update feed URL
└── package.json                # electron-updater dependency
```

## Configuration

### `electron-builder.yml` — Publish Block

```yaml
# frontend/dekstop/electron-builder.yml:41-43
publish:
  provider: generic
  url: https://example.com/auto-updates
```

The `publish` section is read by `electron-builder` at package time and injected into the built application so that `electron-updater` knows where to look at runtime.

### `dev-app-update.yml` — Development Override

```yaml
# frontend/dekstop/dev-app-update.yml:1-3
provider: generic
url: https://example.com/auto-updates
updaterCacheDirName: echo-updater
```

Used during `electron-vite dev` to provide a local update feed URL. Without this file, the dev app would have no `publish` config and skip update checks.

### Update Feed Format

The server at `https://example.com/auto-updates/` must serve:
- `latest.yml` (Windows) or `latest-mac.yml` / `latest-linux.yml` — generated by electron-builder
- Update artifacts (`.exe`, `.dmg`, `.AppImage`)

## How Auto-Update Works

1. **Build time**: `electron-builder` writes the `publish` config into `app-update.yml` inside the ASAR
2. **Runtime**: `electron-updater` reads this config and calls `checkForUpdates()`
3. **Update found**: Downloads the new version in the background
4. **User action**: On next quit or explicit `quitAndInstall()`, the app replaces itself and restarts

> **Note:** The current codebase does not wire `electron-updater` into the main process. The dependency and config exist but `autoUpdater.checkForUpdates()` is not called in `src/main/index.ts`. To activate auto-updates, add:
> ```typescript
> import { autoUpdater } from 'electron-updater'
> autoUpdater.checkForUpdatesAndNotify()
> ```
